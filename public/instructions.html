TREASURE HUNT GUIDE

A comprehensive guide for setting up, managing, and executing the staggered loop treasure hunt platform.

Phase 1
Purging and Initializing the Database

Before launching a brand-new game night, you must clear out stale tracking matrices and test clues from previous events to avoid progression overlaps.

  1. Navigate to the management dashboard interface at /admin.
  2. Locate the Game Controls control panel box on the right-hand side.
  3. Click the red Wipe Entire Game Database button to erase old configurations cleanly.
  4. Click the orange Start / Reset Game Progress button to set the platform engine to initial pre-game states.
Phase 2
Configuring Patrol Roster Groups

The platform mathematically spreads groups dynamically based on the exact number and order of patrols registered in the panel.

  1. Under Patrol Setup, input your first team handle name (e.g., Red, Bilby, or Kookaburra).
  2. Assign a distinct color banner theme dropdown option to help the youths identify their dedicated dashboard panel frames instantly.
  3. Click Save Patrol. Repeat this for all active groups participating on the night (supports up to 10 groups).
Phase 3
Building the Staggered Challenge Map

When creating clues, your inputs dictate the physical environment loops. The system matches the pin to the station the kids are physically visiting.

CRITICAL RULE: Clue Step Numbers MUST be entered sequentially starting from 1 without skipping any integers (e.g., 1, 2, 3, 4). Skipping indices breaks the mathematical sequence generator loops.
  1. In the Clue & Challenge Builder, type your sequential step number.
  2. Create a unique 4-digit numeric PIN for that specific station checkpoint.
  3. Fill in the Leader Cheat Sheet Note specifying the precise physical location (e.g., Taped inside the hollow gum tree stump in the courtyard).
  4. Input your clue messages or directions for the kids in the textarea field (Standard HTML, text links, or video markup objects are fully supported here).
  5. Click Save Challenge Step. Your entry will populate the active overview data list instantly.
Core Concept
Understanding the Staggered Route Loops

To avoid bottlenecks and overcrowding, the system offsets team routes on the fly. Each patrol steps through the physical clues in a completely separate order sequence, ensuring groups don't arrive at the same hiding spot simultaneously.

Phase 4
Operations Setup & Fallback Printing

Before launching the hunt out in the dark or around the hall grounds, verify the game state parameters and export your hardcopy sheets.

  1. Look at the top tracker panel. Ensure all patrols are positioned at the Briefing Start node marker.
  2. Click Print Backup Guide / Answer Sheet. This generates a beautifully clean paper sheet containing every unlock code, hiding spot note, and instruction payload. Keep this on your clipboard in case an iPad drops connection or encounters technical issues.
  3. Prepare your physical token code markers (e.g., paper badges, wooden tags, or painted rocks) with the 4-digit codes matching your master list, and hide them out in the designated fields.
Phase 5
Executing Game Night Operations

When dispatching patrols, hand the Patrol Leader their color-coded briefing note containing their unique game launch PIN code. The system matches these starting codes automatically based on registration order:

  • 1st Registered Patrol uses Activation PIN: 1001
  • 2nd Registered Patrol uses Activation PIN: 1002
  • 3rd Registered Patrol uses Activation PIN: 1003
  • 4th Registered Patrol uses Activation PIN: 1004 (And so on up to 1010...)

When the teams type their launch code into the main iPad hub, the software instantly splits them up and delivers their respective separate starting clue instructions. Monitor the Live Patrol Tracking Radar in real-time from your laptop or mobile device to track team locations, monitor progress, and provide hints if an active node box stops moving!

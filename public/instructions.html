Hunt Hub Command Centre Instructions

TREASURE HUNT GUIDE

A comprehensive guide for setting up, managing, and executing the staggered loop treasure hunt platform.

Phase 1
Multi-Game Workspace Management

The system supports standalone game configuration environments (e.g., maintaining a Joey night map separate from a Cub or Scout night course layout) without risk of data corruption or overlapping active session fields.

  1. Navigate to the management dashboard interface at /admin.
  2. Locate the Active Workspace Profiles control panel block.
  3. Select an existing target profile session, or generate a fresh tracking ID string (alphanumeric keys only, spaces are stripped automatically).
  4. Click Switch Workspace. The underlying platform database maps all subsequent operations, routes, logs, and edits to this isolated session row safely.
Phase 2
Purging and Initializing the Active Session

Before launching a new night on your active workspace profile, you must clear out stale tracking matrices and progression timers from previous runs to avoid loop collision bugs.

  1. Ensure the correct profile identity is active in your administration panel context line.
  2. Locate the Game Controls box on the right-hand side.
  3. Click the red Clear Current Workspace Records button to erase old configuration maps and logs cleanly.
  4. Click the orange Start / Reset Game Progress button to initialize patrol tracking counters back to baseline states.
Phase 3
Configuring Patrol Roster Groups

The platform mathematically spreads groups dynamically based on the exact number and registration order of patrols committed to the profile table layout.

  1. Under Patrol Setup, input your team handle name (e.g., Red, Bilby, or Kookaburra).
  2. Assign a distinct color banner theme dropdown option to help the youths identify their dedicated dashboard panel frames instantly.
  3. Click Save Patrol. Repeat this for all active groups participating on the night (supports up to 10 groups).
Phase 4
Building the Challenge Map & Using Magic Write

Your clue inputs dictate the circular physical environmental tracks. The system automatically shifts target pins relative to individual group progression offsets.

CRITICAL SEQUENCING RULE: Clue Step Numbers MUST be entered sequentially starting from 1 without skipping any integers (e.g., 1, 2, 3, 4). Skipping indices breaks the mathematical circuit generator loops.
  1. In the Clue & Challenge Builder, input your sequential step index integer.
  2. Create a unique unlock answer key string for that checkpoint station.
    *Pro-tip: Entering a 4-digit numeric PIN automatically forces the kids' screen to open a numeric keypad input, while word answers display a standard keyboard layout text box.
  3. Fill in the Leader Cheat Sheet Note specifying the precise physical location (e.g., Inside the empty plastic tub near the equipment compound gate).
  4. Use the AI Generation Tool: Instead of typing raw descriptions, select the appropriate target section age group from the dropdown list:
    • Joey Scouts: Produces simple, literal, direct text based on physical tracking markers, colors, and close shapes.
    • Cub Scouts: Produces engaging, active rhyming riddles or playful word puzzles.
    • Scouts: Produces high-level cryptic phrases, tracking references, or wordplay challenges.
  5. Click the **Magic Write 🪄** button. The ultra-low-latency `gemini-3.1-flash-lite` engine uses your Leader Note parameter to instantly generate a tailored, immersive clue block.
  6. Click Save Challenge Step to commit the data record row directly into the database infrastructure.
Core Concept
Understanding the Staggered Route Loops

To avoid bottlenecks and overcrowding, the system offsets team routes on the fly. Each patrol steps through the physical clues in a completely separate order sequence, ensuring groups don't arrive at the same hiding spot simultaneously.

Phase 5
Operations Setup & Fallback Printing

Before launching the hunt out in the dark or around the hall grounds, verify the game state parameters and export your hardcopy sheets.

  1. Look at the top tracker panel. Ensure all patrols are positioned at the Briefing Start node marker.
  2. Click Backup Map Data Layout Configuration (.json) to generate an offline system file of your challenges. You can use the file restore component later to re-ingest the exact same tracking coordinates instantly.
  3. Prepare your physical token code markers (e.g., tags, cards, or painted rocks) matching your master list, and hide them out in the designated fields.
Phase 6
Executing Game Night Operations

When dispatching patrols, hand the Patrol Leader their color-coded briefing note containing their unique game launch PIN code. The system matches these starting codes automatically based on registration order:

  • 1st Registered Patrol uses Activation PIN: 1001
  • 2nd Registered Patrol uses Activation PIN: 1002
  • 3rd Registered Patrol uses Activation PIN: 1003
  • 4th Registered Patrol uses Activation PIN: 1004 (And so on up to 1010...)

When the teams type their launch code into the main iPad hub, the software instantly splits them up and delivers their respective separate starting clue instructions. Monitor the Live Patrol Tracking Radar in real-time from your laptop or mobile device to track team locations, monitor progress, and check duration splits down to the millisecond!
